Apresentação

  • Eventos: Formal Methods Europe
  • International Conference of Z Users series, previously known as the Z User Meeting (ZUM)

Problematização

  • Crescimento dos métodos formais
  • Concepções erradas e equívocos
  • Discernimento sobre a aplicação dos métodos formais
  • Falta de confiança das publicações científicas populares
  • Visão errada de profissionais: exercícios acadêmicos

Formal Methods delay the development process

  • Profissionais capacitados
  • Estimativas de tempo e custo imprecisas
  • Estimativas devem ser baseadas em histórico
  • Histórico existente é formado por domínios específicos
  • Sem familiaridade com domínios
  • Uso de métodos formais pode reduzir tempo de projeto de forma comprovada

Formal Methods are not supported by tools

  • Existem ferramentas para o uso de métodos formais, inclusive até ferramentas livres

Formal Methods mean forsaking traditional engineering design methods

  • Integração dos métodos formais com o ciclo de vida de desenvolvimento de software
  • Foco nas fases de design e especificação
  • Interação entre equipes

Formal Methods only apply to software

  • Uso dos métodos formais para desenvolvimento e verificação de hardware

Formal Methods are not required

  • Métodos formais não são a bala de prata
  • Em muitos casos pode ser excessivo
  • Desejável quando corretude, consistência e completude é uma preocupação
  • Desejável para sistemas "safety-critical"
  • Possibilidade de verificação e validação
  • Necessárias em mais aplicações no futuro

Formal Methods are not supported

  • Orientação a objetos era apenas buzz-word
  • A web tem suporte para metodos formais
  • "Apenas" a Europa estuda métodos formais

Formal Methods people always use Formal Methods

  • Métodos formais não são aplicáveis a todos os segmentos do desenvolvimento
  • Métodos formais pode aumentar o custo financeiro ou de tempo
  • Usado para partes críticas de aplicações
